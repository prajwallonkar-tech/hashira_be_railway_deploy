# Architecture Decision Records (ADRs)

This folder contains Architecture Decision Records for the Hashira backend.

## ADR Template

Create a new file named `YYYY-MM-DD-<short-title>.md` for each decision:

```markdown
# ADR: <Title>

**Date:** YYYY-MM-DD  
**Status:** proposed | accepted | superseded  
**Author:** <name>  
**Jira:** HASH-<ticket> (if applicable)

## Context

What problem or question prompted this decision?

## Decision

What was decided?

## Rationale

Why was this the right choice? What alternatives were considered and rejected?

## Consequences

What are the trade-offs? What becomes easier or harder as a result?
```

## Existing Decisions

Pre-existing architectural decisions are captured in `../trd-summary.md` (Decision Log section).

ADRs in this folder are for decisions made DURING implementation — record anything non-obvious that future developers might question.
